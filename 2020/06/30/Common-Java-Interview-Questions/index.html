强引用、软引用、弱引用、虚引用，是什么，什么时候用

  • StrongReference: 程序代码之中普遍存在的引用赋值，无论如何情况下，只要强引用关系存在，GC就永远不会回收掉被引用的对象。Object obj = new Object()
  • SoftReference: 在系统将要发生OOM之前，会把这些对象列入回收范围内进行第二次回收。若这次回收之后还没有足够的内存，则会抛出内存溢出异常。
  • WeakReference: 被弱引用关联的对象只能生存到下一次垃圾回收之前，when GC is working, whatever the memory is enough, these objects will be collected.
  • PhantomReference: 一个对象是否有虚应用的存在，完全不影响survive time, 也无法通过虚引用来获得一个对象的instance。设置虚引用关联的唯一目的是该对象被回收时，将会收到一个系统通知。
